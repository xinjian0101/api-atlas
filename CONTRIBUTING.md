# Contributing to API Atlas

Read `README.md`, `ABOUT.md`, and `ROADMAP.md` before contributing.

API Atlas should make public API discovery, metadata review, request examples, and availability checks transparent and reproducible.

## Contribution areas

- API metadata and category schemas.
- Search, filtering, favorites, and local collections.
- Request examples and response presentation.
- Availability checks and status history.
- Code-example generation, fixtures, tests, and documentation.

Use an Issue before changing the catalog schema, request model, status-check behavior, or export formats.

Pull requests should include the user problem, intended behavior, non-goals, example metadata, validation steps, compatibility impact, source attribution, and documentation updates.

Use clear commit prefixes such as `feat:`, `fix:`, `docs:`, `test:`, `refactor:`, and `chore:`.

Contributors are responsible for correctness, licensing, and review. Do not publish private endpoints, access information, or API data that cannot be redistributed.

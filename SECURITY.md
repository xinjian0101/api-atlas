# Security Policy

API Atlas is in the foundation phase and has not published a production-ready release.

Do not include private endpoints, account information, or sensitive request data in public Issues. Use GitHub private reporting when available. Otherwise open a short Issue requesting a private communication channel.

A report should identify the affected commit, environment, reproduction steps, observed impact, and a sanitized example.

Changes involving request execution, redirects, local history, imported collections, generated examples, or availability checks require careful review and clear user-facing behavior.

Validated reports will be reviewed privately where practical. Public notes should identify affected versions and corrections without exposing user information.
